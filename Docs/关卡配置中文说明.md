# Battle 关卡配置中文说明

这份给你或其他 AI 快速配置关卡用。当前 Battle 不是 RTS，关卡机制应保持吸血鬼幸存者 like：刷怪、成长、守/冲/打目标，复杂剧情切 AVG 或切场景。

## 主要文件

- 普通测试关：`D:/project-l/scenes/Battle/Battle_00.ini`
- 普通测试场景：`D:/project-l/scenes/Battle/Battle_00.tscn`
- 弹幕/技能测试场景：`D:/project-l/scenes/Battle/Battle_BulletTest.tscn`
- 实体配置：`D:/project-l/BattleAssets/数字ID.json`
- 攻击配置：`D:/project-l/BattleAssets/Attack_xxx.json`
- 装备配置：`D:/project-l/Config/Equipments.csv`

## stage 段

示例：

```ini
[stage]
id="Battle_00"
duration=720
map_size="3200,2133"
background="res://scenes/Battle/Battle_00/Background.png"
block_mask="res://scenes/Battle/Battle_00/Collision.png"
block_alpha_limit=0.1
max_active_entities_soft=72
bio_texture="res://scenes/Battle/Battle_00/Bio.png"
win_condition="destroy_enemy_base"
```

常用字段：

- `id`：关卡 ID。
- `duration`：关卡时长参考。
- `map_size`：地图尺寸。
- `block_mask`：空气墙/不可通行遮罩。
- `block_alpha_limit`：遮罩 alpha 大于该值视为阻挡。
- `max_active_entities_soft`：同屏实体软上限。超过后敌方波次暂缓刷怪，不会删除已有单位。
- `win_condition`：胜利条件。

## 固定实体

```ini
[player]
entity_id="000"
spawn="PlayerSpawn"

[tentacle_base]
entity_id="001"
spawn="TentacleBaseSpawn"
worker_entity_id="020"
worker_spawn_offset="-170,245"

[enemy_base]
entity_id="003"
spawn="EnemyBaseSpawn"
```

- `entity_id` 对应 `BattleAssets/xxx.json`。
- `spawn` 可以是场景里的 spawn 节点名，也可以是坐标或区域规则。
- 工蜂开局由 `worker_entity_id` 指定。

## 胜利条件

当前支持：

- `destroy_enemy_base`：摧毁敌方基地。
- `survive_time`：坚持指定时间。
- `reach_area`：玩家抵达指定区域。
- `escort_entity`：指定 NPC 抵达区域。
- `kill_count`：击杀数量。
- `kill_entity_id`：击杀指定 ID 若干个。
- `hold_area`：在区域内坚持一段时间。
- `activate_areas`：依次/累计激活多个区域。

常配字段在 `[objective]`：

```ini
[objective]
text="在仪式圈内坚持 60 秒"
area="ritual_a"
duration=60
target_count=20
target_entity_id="013"
target_faction="ally"
areas="a|b|c"
```

不是每个胜利条件都需要全部字段。

## 区域配置

圆形区域：

```ini
[area_ritual_a]
shape="circle"
center="1600,900"
radius=180
visible=true
```

矩形区域：

```ini
[area_exit]
shape="rect"
center="2800,1600"
size="220,180"
visible=true
```

区域可用于：

- 胜利条件。
- 事件触发。
- 刷怪位置。
- 简单视觉提示。

## 敌人波次

示例：

```ini
[wave_001]
time=2
entity_id="002"
count=14
spawn="EnemySpawnA"
random_radius_min=40
random_radius=210
interval=0.35
ai_mode="chase_nearest"
movement_mode="direct"
target_factions="player|tentacle"
target_priority_order="building|player|minion|unit"
```

字段说明：

- `time`：第几秒开始。
- `entity_id`：刷哪个实体。
- `count`：刷几个。
- `spawn`：刷怪位置。
- `random_radius_min/random_radius`：围绕刷怪点随机范围。
- `interval`：每个怪之间的间隔。
- `ai_mode`：常用 `chase_nearest`。
- `movement_mode`：常用 `direct`、`strafe_close`、`orbit_close`。
- `target_factions`：敌人要找谁。
- `target_priority_order`：优先打什么。

`spawn` 可写：

- 场景节点名：`EnemySpawnA`
- 坐标：`1200,800`
- 玩家周围：`random_around_player`
- 我方基地附近：`random_near_tentacle_base`
- 区域内：`area:ritual_a`

## 关卡事件

事件段名必须以 `event_` 开头：

```ini
[event_intro]
condition=time
time=3
action=play_avg
avg_id="battle_intro_001"
once=true
```

支持条件：

- `time`：到达时间。
- `player_enter_area`：玩家进入区域。
- `kill_count`：击杀数达到 `target_count`。
- `win`：战斗胜利。
- `loss`：战斗失败。
- `player_hp_below`：玩家血量比例低于 `amount`。
- `base_hp_below`：我方基地血量比例低于 `amount`。
- `enemy_base_hp_below`：敌方基地血量比例低于 `amount`。

支持动作：

- `play_avg`：播放 AVG。
- `spawn_entity`：刷实体。
- `add_base_bio`：给基地加生物质。
- `heal_player`：治疗玩家。
- `heal_base`：治疗基地。
- `damage_enemy_base`：伤害敌方基地。
- `win`：直接胜利。
- `loss`：直接失败。
- `area_fx`：播放区域提示。
- `change_scene`：切换场景。

多个动作可以用 `|`：

```ini
[event_after_win]
condition=win
action=play_avg|change_scene
avg_id="after_battle_001"
scene_path="res://scenes/SomeNextScene.tscn"
once=true
```

低血事件例子：

```ini
[event_base_danger]
condition=base_hp_below
amount=0.35
action=play_avg|heal_base
avg_id="base_danger"
amount=400
once=true
```

注意：同一个事件里 `amount` 目前既可被条件用，也可被动作使用。复杂事件建议拆成两个事件，避免语义混乱。

## 实体 JSON 快速规则

实体文件在 `BattleAssets/xxx.json`。

基础结构：

```json
{
  "id": "024",
  "name": "魅惑同化触",
  "type": "minion",
  "faction": "tentacle",
  "tags": ["tentacle", "minion"],
  "visual": {"gif": "res://BattleAssets/024.gif", "max_size": [100, 140], "offset": [0, 0]},
  "body": {"radius": 24, "sense_radius": 660},
  "stats": {"max_hp": 240, "move_speed": 118, "attack": 8, "defense": 3, "hp_regen": 0},
  "ai": {"mode": "chase_nearest", "target_factions": ["enemy"]},
  "attacks": []
}
```

视觉注意：

- 基地图片中心点就是实体圆心/碰撞圆心。
- `visual.max_size` 可限制 GIF/PNG 最大显示大小。
- `visual.offset` 只在确实需要偏移时使用，基地不要乱偏移。

## 攻击 JSON 快速规则

攻击文件在 `BattleAssets/Attack_xxx.json`。

常用字段：

- `kind="attack_instance"`：现代攻击系统。
- `origin`：攻击从哪里出现。
- `aim`：朝向。
- `emitter`：数量/散射。
- `motion`：运动。
- `hit_shape`：判定形状。
- `hit_rule`：命中规则。
- `target_filter`：目标过滤。
- `effects`：伤害/治疗/状态/生成子攻击。
- `visual`：素材和颜色。

近战素材锚点：

```json
"visual": {
  "texture": "res://BattleAssets/StrongMelee.png",
  "anchor": "left_center"
}
```

含义：默认图向右，图片左边中心贴角色攻击点；角色向左时攻击旋转后等价于镜像绑定。

## 装备配置

装备在 `Config/Equipments.csv`。

格式：

```csv
id,name,unlocked,icon,description,effect_keys
E007,小皮鞭,TRUE,,连续移动积累最多3层蓄力；停止时释放多段斩，其他常规武器伤害下降。,whip_charge|whip_weapon_penalty
```

已支持的重点 effect keys：

- `whip_charge`
- `whip_weapon_penalty`
- `range_weapon_only`
- `bind_area_boost`
- `liquid_madness`
- `forced_forward_move`
- `player_speed_up_small`
- `player_hp_down_big`
- `exhibitionist`
- `melee_lifesteal`
- `skill2_full_mana_required`
- `mana_recovery_up`
- `crit_rate_down`
- `minute_lust_add`
- `base_bio_cycle_yield_up`

新装备建议：

1. 先写 CSV 行。
2. 能复用攻击就新增 `Attack_equipment_xxx.json`。
3. 在 `BattleDirector.gd` 的装备事件函数里接入。
4. 更新 `Docs/EQUIPMENT_MVP.md`。

## 性能建议

- 先用 `max_active_entities_soft=72` 控制同屏数量。
- 想提高压力，不要只加数量，优先提高怪物血量、攻击、速度、波次节奏。
- 少用超多 GIF 和长寿命多段攻击。
- 大范围特效尽量用一个 AttackInstance，不要每 tick 生成大量新节点。
