# Battle 属性公式与平衡笔记

## 推荐总公式

多数可平衡数值按这个方向统一：

```text
最终值 = (基础值 + 局外固定加值 + 局内固定加值)
       * (1 + 局外比例加成 + 局内比例加成)
       * 特殊乘区
```

这个公式适合生命、攻击、移速、生命恢复、Mana 回复、生物质产量等常规属性。

## 为什么保留特殊乘区

所有比例都塞进同一个池会让爆发技能和特色装备变得不明显。建议保留这些独立乘区：

- 角色/武器类型乘区：例如近战、远程、范围的角色适配。
- 局内普通比例池：升级、临时局内数值强化。
- 局外普通比例池：局外升级、长期成长。
- 技能/状态乘区：短时间爆发、无敌、过载、减速等。
- 目标类型乘区：对建筑、对杂鱼、对精英的修正。
- 暴击乘区：只给 player、敌我小兵；基地类默认不暴击。

## 当前系统与公式的对应

- `Characters.csv`：角色基础倍率，当前在进战斗时作用到玩家。
- `WeaponUpgrades.csv`：武器局内升级，当前会直接改玩家已有攻击字典。
- `RunStatUpgrades.csv`：局内通用属性升级，当前有固定加值、暴击、Mana 回复等。
- `Equipments.csv.effect_keys`：装备机制入口，当前能改血量/回血/Mana/暴击/淫能/基地生物质产量/技能限制。
- 基地与小兵升级：当前在 `BattleDirector.BASE_LEVEL_CHOICES` 内直接改基地、小兵或队列参数。

## 后续应收敛的方向

- 建一个轻量属性汇总层，把固定加值、普通比例、特殊乘区分清楚。
- 不急着大重构；优先在新增机制时按同一命名和乘区思路写。
- 对基地类攻击保持 `crit_chance = 0`、无暴击倍率。
- 对装备保持“限制 x 收益”的体验，不和局外升级混成纯正向数值。
- 对局外升级保持稳定正向，尽量只进固定值或普通比例池。

## 平衡时要看的核心指标

- 玩家有效输出：攻击、频率、范围、暴击、技能倍率。
- 玩家安全性：最大生命、生命恢复、移速、无敌/复活类机制。
- 基地压力：基地生命、回血、自保圈、生产生物质、出兵节奏。
- 小兵贡献：产出成本、产出周期、目标优先级、对建筑效率、杂鱼清理能力。
- 战后收益：淫能固定加值、倍率、风险装备带来的收益曲线。
